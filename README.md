<h2>1. 개요 </h2>
    <b>리눅스(Linux)</b>는 오픈 소스 기반의 운영체제로, 1991년 리누스 토르발스(Linus Torvalds)가 처음 개발을 시작했습니다. 리눅스는 다양한 하드웨어에서 구동 가능하며, 서버, 데스크탑, 모바일, 임베디드 시스템 등 다양한 환경에서 사용됩니다. 특히 서버 운영체제로 널리 사용되고 있으며, 안정성과 보안성이 뛰어나 기업 및 개발자들에게 인기가 많습니다. 수많은 배포판(우분투, 센토스, 페도라 등)을 통해 제공되며, 각 배포판은 사용자 요구에 맞춘 다양한 기능을 지원합니다.
리눅스는 커널을 중심으로 구성되며, 다양한 소프트웨어 패키지를 사용하여 시스템을 구성할 수 있습니다. 오픈 소스 철학을 기반으로 누구나 소스 코드를 자유롭게 수정하고 재배포할 수 있다는 점이 큰 장점입니다.
<h2>2. 라이센스 </h2>
리눅스는 GPL(General Public License) 버전 2.0 하에서 배포됩니다. 

GNU GPL은 다음과 같은 핵심 원칙을 포함합니다:
<ul>
   <li> 소프트웨어의 소스 코드를 공개해야 한다.</li>
   <li> 소프트웨어를 수정하고 재배포할 수 있다.</li>
   <li> 수정된 소프트웨어도 동일한 라이선스 하에 배포해야 한다.</li>
</ul>

이로 인해 리눅스는 누구나 무료로 사용할 수 있으며, 상업적 용도로도 사용할 수 있습니다. 다만, 상업적으로 판매된 경우에도 소스 코드는 공개되어야 하며, 사용자에게 무료로 제공될 수 있는 권한이 보장됩니다.
리눅스의 상업적 배포판(예: 레드햇 엔터프라이즈 리눅스, SUSE Linux Enterprise 등)은 사용자의 지원 서비스나 특정 기능을 포함하며, 상업용 라이선스를 제공하기도 합니다. 
리눅스는 전 세계의 수많은 개발자들이 기여하고 협력하는 커뮤니티 기반 프로젝트입니다. 개발자들은 코드 작성, 버그 수정, 문서 작성 등 다양한 방식으로 기여할 수 있으며, 커뮤니티의 적극적인 참여 덕분에 리눅스 커널이 지속적으로 발전하고 있습니다.

  - 소스 코드 자유 수정 및 재배포 가능

  - 특정 하드웨어 환경에 최적화하거나, 보안·성능 강화 가능

   - **특징**

   - **사용자층**

   - **특징**

   - **사용자층**  
     안정성이 중요한 서버 운영 및 기업 환경
   - **예시**  
     웹 호스팅 회사에서 수백 개의 웹사이트를 운영하기 위한 서버 관리에 사용되며, 안정성과 보안을 중시하는 서버 환경에 적합합니다.

<h3> 패키지 관리 시스템 </h3>
리눅스(Linux) 운영체제에서 소프트웨어를 설치, 업데이트, 제거하는 과정은 패키지 관리 시스템에 의해 관리됩니다. 패키지 관리 시스템은 패키지라 불리는 소프트웨어 묶음을 다운로드하고 설치하며, 의존성을 자동으로 해결해주는 도구입니다. 이를 통해 사용자는 소프트웨어 설치 및 유지보수를 보다 쉽게 할 수 있습니다. 대표적인 리눅스 패키지 관리 도구에는 <b>APT</b>, <b>YUM</b>, <b>DNF</b> 등이 있습니다.

#### 1. APT (Advanced Package Tool)
APT는 주로 <b>데비안(Debian)</b> 및 <b>우분투(Ubuntu)</b> 계열의 리눅스 배포판에서 사용됩니다. APT는 사용자에게 직관적인 명령어 인터페이스를 제공하며, 소프트웨어 설치뿐만 아니라 의존성 해결, 업데이트, 삭제 작업을 쉽게 할 수 있습니다.

- **설치 예시**:
  ```bash
  sudo apt update  # 패키지 목록 업데이트
  sudo apt install 패키지명  # 패키지 설치
  
- **APT의 특징**:
  - 의존성을 자동으로 해결해준다.
  - 명령어 구조가 사용자 친화적이다.
  - 광범위한 소프트웨어 저장소를 지원한다.

#### 2. YUM (Yellowdog Updater, Modified)

**YUM**은 **RHEL(Red Hat Enterprise Linux)**, **CentOS** 등의 배포판에서 주로 사용되는 패키지 관리 도구입니다. YUM은 RPM 기반의 패키지들을 관리하며, 네트워크를 통해 원격 저장소에서 패키지를 다운로드하고 설치할 수 있습니다. 또한, 패키지의 의존성을 자동으로 해결하여 설치 과정을 단순화합니다.

- **설치 예시**:
  ```bash
  sudo yum install 패키지명
  ```
- **YUM의 특징**:
  - 원격 저장소에서 패키지 설치가 가능하다.
  - 의존성 관리 및 자동으로 해결한다.
  - 레드햇 계열 배포판에서 널리 사용한다.


#### 3. DNF (Dandified YUM)
**DNF**는 YUM의 후속 버전으로, **페도라(Fedora)** 및 **RHEL 8** 이후의 배포판에서 사용됩니다. DNF는 YUM에 비해 성능이 개선되고 의존성 충돌 처리 능력이 향상되었습니다. 또한, 메타데이터 처리 방식이 개선되어 YUM보다 더 빠르고 안정적인 패키지 관리가 가능합니다.

- **설치 예시**:
  ```bash
  sudo dnf install 패키지명
  ```

- **DNF의 특징**:
  - 더 나은 성능 및 메모리 관리를 가능하게 한다.
  - YUM에 비해 의존성 처리 및 충돌 해결 기능을 향상시킨다.
  - 개선된 CLI 명령어 경험을 제공한다
 
#### 4. RPM (Red Hat Package Manager)
**RPM**은 RHEL, CentOS, Fedora 등에서 사용하는 기본 패키지 형식이자 패키지 관리 도구입니다. RPM은 단일 패키지 파일을 설치, 삭제 및 관리할 수 있으며, 네트워크 연결이 없는 상태에서도 로컬에서 직접 패키지를 설치할 수 있습니다. 그러나 RPM 자체는 의존성 관리를 하지 않으며, 이를 보완하기 위해 YUM이나 DNF가 사용됩니다.

- **설치 예시**:
  ```bash
  sudo rpm -ivh 패키지명.rpm  # 패키지 설치
  sudo rpm -e 패키지명  # 패키지 제거
  ```

- **RPM의 특징**:
  - 개별 패키지 관리
  - 패키지 설치 및 삭제 기능
  - 의존성 해결 기능이 없음 (YUM, DNF와 병행 사용)

#### 5. 패키지 관리 시스템의 주요 기능
패키지 관리 시스템은 단순히 소프트웨어 설치에만 그치지 않고 다양한 기능을 제공합니다.

- **패키지 업데이트**: 시스템의 모든 패키지를 최신 상태로 유지할 수 있습니다.
  ```bash
  sudo apt upgrade  # APT 예시
  sudo dnf upgrade  # DNF 예시
  ```
  
- **의존성 관리**: 패키지 설치 시 해당 패키지가 의존하는 다른 소프트웨어를 자동으로 설치합니다. 예를 들어, 프로그램 A를 설치하려면 프로그램 B와 C가 필요한 경우, 패키지 관리 시스템은 A, B, C를 모두 설치해줍니다.

- **저장소 관리**: 다양한 소프트웨어 저장소를 추가하거나 제거할 수 있습니다. 각 배포판은 기본적으로 공식 저장소를 제공하며, 사용자는 필요에 따라 서드파티 저장소를 추가할 수 있습니다.
  ```bash
  sudo add-apt-repository ppa:저장소명  # APT 예시
  ```

- **캐시 관리**: 패키지 관리 도구는 다운로드한 패키지를 캐시에 저장합니다. 이 캐시를 삭제하여 디스크 공간을 절약할 수 있습니다.
  ```bash
  sudo apt clean  # APT 예시
  ```

#### 6. 패키지 관리 도구의 중요성

패키지 관리 도구는 시스템을 안전하고 최신 상태로 유지하는 데 필수적입니다. 이를 통해 사용자는 손쉽게 최신 보안 패치를 적용하고, 필요한 소프트웨어를 안정적으로 설치할 수 있습니다. 다양한 배포판마다 각각의 패키지 관리 도구가 있지만, 그 목적과 기본 기능은 비슷합니다. 사용자는 자신이 사용하는 배포판에 맞는 도구를 익혀두는 것이 중요합니다.

<h3> 파일 시스템과 권한 관리 </h3>
리눅스 파일 시스템은 *트리 구조로 되어 있으며, 루트 디렉터리(/)가 최상단에 위치하고 그 아래에 다양한 디렉터리가 연결됩니다. 파일과 디렉터리는 사용자(user), 그룹(group), 기타 사용자(others)로 나뉘어 권한을 관리합니다. 권한은 읽기(read), 쓰기(write), 실행(execute)으로 나뉘며, chmod 명령어로 권한을 설정하고, chown으로 소유자 및 그룹을 변경할 수 있습니다. 또한 SUID, SGID, Sticky Bit 같은 특수 권한도 존재하여 보안을 강화합니다.<br/><br/>

#### 리눅스 파일 시스템 주요 특징
1. 모든 것이 파일: 장치, 프로세스 등 모든 자원이 파일로 표현됩니다.
2. 계층적 구조: 루트(/)부터 시작해 파일과 디렉터리가 트리 구조로 연결됩니다.
3. 저널링: 대부분의 리눅스 파일 시스템은 저널링을 통해 데이터 무결성을 보장합니다.
4. 다양한 파일 시스템 지원: ext4, Btrfs, XFS 등 여러 파일 시스템을 지원하며 사용자의 필요에 맞게 선택할 수 있습니다.<br/><br/>

주요 디렉터리는 다음과 같습니다:<br/>
/bin: 기본 명령어가 저장된 디렉터리로, 시스템 부팅과 관련된 필수 파일이 포함됩니다.  
/home: 사용자별 홈 디렉터리. 각 사용자는 고유한 홈 디렉터리를 할당받아 개인 파일을 저장합니다.  
/etc: 시스템 설정 파일들이 포함된 디렉터리.  
/var: 로그 파일 및 가변 데이터를 저장하는 디렉터리.  
/tmp: 임시 파일을 저장하는 디렉터리로, 재부팅 시 내용이 삭제됩니다.  

*[Tree](https://namu.wiki/w/%ED%8A%B8%EB%A6%AC(%EA%B7%B8%EB%9E%98%ED%94%84), "What is Tree?") : 트리에 대한 설명

리눅스는 파일을 단순한 데이터의 모음이 아니라, 모든 것이 파일이라는 철학을 따릅니다. 장치나 소켓조차 파일로 취급되어, 이를 통해 장치에 쉽게 접근하고 제어할 수 있습니다.

### 파일 권한 관리
리눅스는 사용자(user), 그룹(group), 그리고 기타 사용자(others)로 나뉘어 파일에 대한 접근 권한을 관리합니다. 각 파일에는 소유자와 그룹이 할당되어 있으며, 이를 통해 파일 접근 권한이 설정됩니다. 
파일 권한은 세 가지로 구분됩니다:

- 읽기 권한(read, r): 파일 내용을 읽을 수 있는 권한.
- 쓰기 권한(write, w): 파일 내용을 수정하거나 삭제할 수 있는 권한.
- 실행 권한(execute, x): 파일을 실행할 수 있는 권한.
  
파일의 권한 설정은 ls -l 명령어를 통해 확인할 수 있으며, 출력 결과는 다음과 같은 형식으로 표시됩니다: 

```diff
  -rwxr-xr--
```

여기서 각 문자는 해당 권한을 나타내며, 앞의 3자리(rwx)는 파일 소유자의 권한, 중간의 3자리(r-x)는 그룹의 권한, 마지막 3자리(r--)는 기타 사용자의 권한을 의미합니다.

#### 권한 수정
파일의 권한은 chmod 명령어를 사용해 수정할 수 있습니다. 예를 들어, 파일에 대해 읽기, 쓰기, 실행 권한을 모두 부여하려면 다음과 같은 명령을 사용할 수 있습니다:

```bash
chmod 755 파일명
```

숫자 755는 8진수로 권한을 나타내며, 각 자리는 소유자, 그룹, 기타 사용자에 대한 권한을 의미합니다. 이 숫자들은 다음과 같이 계산됩니다:
따라서 755는 소유자는 모든 권한을(7=4+2+1), 그룹과 기타 사용자는 읽기 및 실행 권한만을(5=4+1)을 갖는다는 의미입니다.

- 읽기 권한 : 4
- 쓰기 권한 : 2
- 실행 권한 : 1

#### 소유자 및 그룹 변경
파일의 소유자와 그룹은 chown 명령어를 통해 변경할 수 있습니다. 예를 들어, 파일의 소유자를 user1, 그룹을 group1로 변경하려면 다음과 같은 명령을 사용합니다:

```bash
chown user1:group1 파일명
```

### 특수 권한
리눅스는 파일과 디렉터리에 적용할 수 있는 몇 가지 특수 권한을 지원합니다:

- SUID (Set User ID): 실행 파일이 소유자의 권한으로 실행되도록 설정하는 권한.
- SGID (Set Group ID): 실행 파일이 그룹 소유자의 권한으로 실행되도록 설정하는 권한. 디렉터리에서 SGID가 설정되면, 새로 생성된 파일은 해당 디렉터리의 그룹 소유자가 자동으로 할당됩니다.
- Sticky Bit: 디렉터리에서 설정할 경우, 파일 삭제 권한을 파일 소유자에게만 부여하는 권한.
  
이 특수 권한들은 보안 및 접근 제어에 중요한 역할을 합니다.

#### 권한 관리의 중요성
리눅스는 권한 관리가 엄격하게 이루어지며, 다중 사용자 환경에서 보안을 강화합니다. 이로 인해 시스템의 무결성을 보호하고 불필요한 데이터 접근을 제한할 수 있습니다.

리눅스 파일 시스템과 권한 관리는 서버 환경, 특히 웹 서버나 데이터베이스 서버처럼 여러 사용자가 접근하는 시스템에서 필수적인 요소입니다.
